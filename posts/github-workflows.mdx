GitHub workflows can be triggered via the API or GitHub Actions. GitHub Actions allow you to automate tasks directly from your GitHub repository, while the API provides programmatic access to your repositories and various tasks that can be performed on them.

By utilizing these workflows, you can streamline your development process, reduce manual intervention, and improve overall productivity.

## Workflow actions

1. Create a repository
2. Create a branch
3. Create a file
4. Edit a file
5. Create a commit
6. Create an issue
7. Create a pull request
8. Merge a pull request
9. Create a project
10. Create a tag
11. Create a release
